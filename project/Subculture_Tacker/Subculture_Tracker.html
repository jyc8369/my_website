Subculture Tracker

Wuthering Waves와 Arknights Endfield의 가챠 기록을 자동 수집해 공통 JSON으로 변환하고, 웹 UI로 확인하는 도구의 개발 문서입니다.

Overview Structure Run GitHub 목록으로

Overview

Subculture Tracker는 로컬 게임 로그 및 캐시 파일에서 토큰과 URL을 추출하고, 게임 서버 API를 호출해 가챠 기록을 수집하는 Windows 기반 도구입니다.

Supported Games Wuthering Waves(`wuwa`)와 Arknights Endfield(`endfield`)를 지원합니다.
Output 게임별 데이터를 공통 JSON 형식으로 저장합니다.
UI Flask + pywebview 또는 브라우저 모드로 결과를 확인합니다.

Development Environment

Python Python 3.7 이상
Flask `Flask>=3.0`
HTTP `requests>=2.0`
Parsing `beautifulsoup4>=4.0`
Desktop UI `pywebview>=3.0`

Run

Basic python run.py로 실행합니다.
Dev Browser python run.py -dev로 브라우저 모드를 사용합니다.
Virtualenv PowerShell 기준 python -m venv .venv 후 .\.venv\Scripts\Activate.ps1로 활성화합니다.
Install pip install -r requirements.txt로 의존성을 설치합니다.

Runtime Flow

Admin Re-run Windows에서 관리자 권한이 아니면 자동으로 관리자 권한 재실행을 시도합니다.
Console Hide Windows에서 pythonw.exe를 재실행해 콘솔 표시를 줄입니다.
Logging 실행 시 lastlog.log를 생성해 앱 로그를 기록합니다.
Frontend backend.web.run(use_browser=dev_mode)로 Flask와 UI를 함께 구동합니다.

run.py 진입점, 권한 확인, Flask 서버 시작을 담당합니다.
backend/ 실제 수집과 웹 UI 로직을 분리합니다.
web/ 정적 HTML, CSS, JS 리소스가 위치합니다.
data/ 수집 결과 JSON과 설정 파일을 저장합니다.

Backend Details

backend/web.py Flask 앱 초기화, API 엔드포인트, 작업 큐, 백그라운드 워커를 관리합니다.
backend/wuwa.py 로그에서 마지막 가챠 기록 URL을 찾고 서버 API를 호출해 데이터를 수집합니다.
backend/endfield.py 캐시 파일에서 토큰을 추출하고 Endfield 기록을 페이징 수집합니다.
backend/webview.py pywebview 네이티브 창을 생성합니다.

Data Model

Wuwa Output data/wuwa_<profilename>.json 형식으로 저장합니다.
Endfield Output data/endfield_<profilename>.json 형식으로 저장합니다.
Schema id와 banners를 중심으로 공통 스키마를 사용합니다.
Dedup 동일 ID, 토큰 기준의 이전 결과 파일은 중복 제거합니다.
{
  "id": "user-or-token",
  "banners": [
    {
      "bannerName": "풀 이름",
      "items": [
        {
          "type": 1,
          "seqId": "123456",
          "name": "캐릭터 이름",
          "rarity": "5",
          "timestamp": 1680000000000
        }
      ]
    }
  ]
}

Debug & Logs

lastlog.log 백엔드 로그가 기록됩니다.
Unified Logging Flask 요청 로그와 werkzeug 로그를 같은 로거로 출력합니다.
Error Tracking HTTP 요청 실패, JSON 파싱 오류, 파일 I/O 오류를 기록합니다.

Packaging

Build Script build-exe.ps1로 PyInstaller 빌드를 수행합니다.
Spec SCT.spec을 사용해 단일 실행 파일을 생성합니다.
Options -Clean은 이전 결과를 삭제하고, -NoUpx는 UPX 압축을 생략합니다.
Output 빌드된 결과물은 dist/에 생성됩니다.

Additional Notes

Profiles profilename은 결과 파일명에 직접 쓰이므로 UI에서 충돌에 주의해야 합니다.
Export API /exporter는 동기 wait와 비동기 작업 큐를 모두 지원합니다.
Config data/에는 web-setting.json 같은 설정 파일도 저장될 수 있습니다.
